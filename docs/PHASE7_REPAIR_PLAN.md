# Phase 7 — Multi-Tenant Repair Platform

Planned scope:

- Tenant-generated repair IDs for every shop
- Mahar Shwe external Repair ID import through the existing repair-tracking API
- Explicit provider/source linkage for partner-shop repair handoff
- Device-level history keyed by normalized IMEI or serial fingerprint
- Immutable repair timeline events
- Multi-tenant isolation and duplicate-import protection
- Advanced Repair workspace with import, new intake, history and status tracking
- Suppliers page remains deferred until purchase and payable workflows require it
